# Plan 03-03: Stripe Webhook for Payment Confirmation

## Context

Currently, the checkout flow:
1. Creates WooCommerce order (status: pending)
2. Creates Stripe PaymentIntent
3. Confirms card payment client-side
4. On success, client calls `POST /api/update-order` to mark order as "processing"

**The problem**: Step 4 is client-side. If the browser closes, crashes, or the network drops after payment is confirmed but before the update call completes, the WooCommerce order stays "pending" forever even though Stripe charged the customer.

**The solution**: A server-side Stripe webhook that listens for `payment_intent.succeeded` events and updates the WooCommerce order status automatically. This is the industry-standard approach.

**How to link payment to order**: The Stripe PaymentIntent `metadata` field must carry the WooCommerce order ID. The checkout flow must be updated to pass `metadata: { wc_order_id: order.id }` when creating the PaymentIntent.

## Tasks

### Task 1: Add webhook route to payments.cjs

The Stripe webhook requires the **raw request body** (not parsed JSON) for signature verification. This means the webhook route must be registered BEFORE express.json() middleware, or we need a special raw body parser for that route.

In `server/routes/payments.cjs`, add the webhook endpoint:

```js
router.post('/stripe-webhook', express.raw({ type: 'application/json' }), async (req, res) => {
  const sig = req.headers['stripe-signature'];
  const webhookSecret = process.env.STRIPE_WEBHOOK_SECRET;

  if (!webhookSecret) {
    console.error('STRIPE_WEBHOOK_SECRET not set');
    return res.status(500).send('Webhook secret not configured');
  }

  let event;
  try {
    event = stripe.webhooks.constructEvent(req.body, sig, webhookSecret);
  } catch (err) {
    console.error('Webhook signature verification failed:', err.message);
    return res.status(400).send(`Webhook Error: ${err.message}`);
  }

  if (event.type === 'payment_intent.succeeded') {
    const paymentIntent = event.data.object;
    const wcOrderId = paymentIntent.metadata?.wc_order_id;

    if (wcOrderId) {
      try {
        const api = require('../woocommerce.cjs');
        await api.put(`orders/${wcOrderId}`, {
          status: 'processing',
          transaction_id: paymentIntent.id,
        });
        console.log(`Order ${wcOrderId} marked processing via webhook`);
      } catch (err) {
        console.error(`Failed to update order ${wcOrderId}:`, err.message);
        // Return 200 to prevent Stripe from retrying — log the error but don't fail
      }
    }
  }

  res.json({ received: true });
});
```

Note: `express.raw()` is built into Express — no new dependency needed.

### Task 2: Update create-payment-intent to accept and store metadata

Modify the `/create-payment-intent` route to accept `wcOrderId` in the request body and include it in PaymentIntent metadata:

```js
router.post('/create-payment-intent', async (req, res) => {
  const { amount, currency = 'usd', wcOrderId } = req.body;
  const paymentIntent = await stripe.paymentIntents.create({
    amount,
    currency,
    automatic_payment_methods: { enabled: true },
    metadata: wcOrderId ? { wc_order_id: String(wcOrderId) } : {},
  });
  res.json({ clientSecret: paymentIntent.client_secret });
});
```

```ts
export const createPaymentIntent = async (amount: number, wcOrderId?: number) => {
    try {
        const amountInCents = Math.round(amount * 100);
        const response = await axios.post(`${API_URL}/create-payment-intent`, {
          amount: amountInCents,
          wcOrderId
        });
        return response.data;
    } catch (error) {
        console.error('Error creating payment intent:', error);
        throw error;
    }
};
```

### Task 4: Register webhook route in server/index.cjs

The `/api/stripe-webhook` route uses `express.raw()` for the body, so it MUST be registered before the global `express.json()` middleware. Check `server/index.cjs` to see middleware order and ensure the payments router is registered correctly.

If the global middleware order is: `app.use(express.json())` then `app.use('/api', paymentsRouter)`, the raw body override in the route itself (`express.raw({ type: 'application/json' })`) should work because route-level middleware overrides global middleware for that specific route.

However, to be safe, explicitly add a route-level raw body override as shown in Task 1's code. This is the standard pattern Stripe recommends.

Add `STRIPE_WEBHOOK_SECRET` to the env var list in comments/documentation in `server/index.cjs` if there's an env check section.

## Environment Variables Required

The following env vars must be set for the webhook to work in production:
- `STRIPE_WEBHOOK_SECRET` — from Stripe Dashboard > Webhooks > Signing secret

For local development with Stripe CLI:
```bash
stripe listen --forward-to localhost:3001/api/stripe-webhook
```
The CLI will print a webhook secret starting with `whsec_` — set this as `STRIPE_WEBHOOK_SECRET` in `.env.local`.
