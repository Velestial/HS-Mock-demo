# Plan 03-02: Fix Payment Amount and Production Turnstile

## Context

Two payment hardening issues:

**1. Stripe amount bug** — `createPaymentIntent` in `services/api.ts` sends the dollar amount directly to the server, but Stripe expects amount in cents (integer). So a $42.99 order creates a Stripe PaymentIntent for $0.43 (43 cents) instead of $42.99.

The server's `payments.cjs` passes amount straight to Stripe:
```js
const paymentIntent = await stripe.paymentIntents.create({ amount, ... });
```

The fix is to convert dollars → cents before sending. The conversion should happen in `services/api.ts` (the client-facing API layer) so the server stays simple.

**2. Hardcoded test Turnstile key** — `components/CheckoutPage.tsx` has:
```tsx
siteKey="1x00000000000000000000AA"
```
This is the Cloudflare test key (always passes, no bot protection). It must read from `import.meta.env.VITE_TURNSTILE_SITE_KEY` so the production key can be injected via environment.

Also: the same hardcoded test key is used in `components/LoginPage.tsx` and `components/RegisterPage.tsx` (if they have Turnstile). Check and fix those too.

## Tasks

### Task 1: Fix createPaymentIntent to send cents

In `services/api.ts`, update `createPaymentIntent`:

```ts
export const createPaymentIntent = async (amount: number) => {
    try {
        const amountInCents = Math.round(amount * 100);
        const response = await axios.post(`${API_URL}/create-payment-intent`, { amount: amountInCents });
        return response.data;
    } catch (error) {
        console.error('Error creating payment intent:', error);
        throw error;
    }
};
```

### Task 2: Fix Turnstile siteKey in CheckoutPage

In `components/CheckoutPage.tsx`, find the Turnstile widget:
```tsx
siteKey="1x00000000000000000000AA"
```

Replace with:
```tsx
siteKey={import.meta.env.VITE_TURNSTILE_SITE_KEY || '1x00000000000000000000AA'}
```

This keeps the test key as fallback for local dev without the env var set, while allowing production to inject the real key.

### Task 3: Fix Turnstile in LoginPage and RegisterPage

Search for `siteKey` in `components/LoginPage.tsx` and `components/RegisterPage.tsx` (if exists). Apply same pattern:
```tsx
siteKey={import.meta.env.VITE_TURNSTILE_SITE_KEY || '1x00000000000000000000AA'}
```

Check `components/RegisterPage.tsx` exists (the form may be inline in `LoginPage.tsx`). Look for any hardcoded Turnstile keys and fix them all.
