<task type="auto">
  <name>Task 1: Create utils/analytics.ts — GA4 init and typed tracking functions</name>
  <files>utils/analytics.ts</files>
  <action>
Create `utils/analytics.ts` at the project root (alongside App.tsx, not inside a subfolder). This file is the single GA4 integration point for the entire app.

Add the following exports:

1. `initGA(): void` — Injects the gtag.js CDN script and initializes GA4.
   - Guard: if `!GA_ID || typeof window === 'undefined'` return early (no crash when env var missing).
   - Guard: if `window.gtag` already exists, return early (prevents double-inject on hot reload).
   - Inject `<script async src="https://www.googletagmanager.com/gtag/js?id=${GA_ID}">` into `document.head`.
   - CRITICAL: use `window.gtag = function() { window.dataLayer.push(arguments); }` — NOT an arrow function, NOT spread. This is the most common GA4 failure. Arrow functions create true Arrays; GA4 requires the `arguments` object (array-like).
   - Call `window.gtag('js', new Date())`.
   - Call `window.gtag('config', GA_ID, { send_page_view: false })` — false prevents double page_view. App.tsx fires the first page_view manually via useEffect.

2. `trackPageView(path: string, title: string): void`
   - Guard: `if (!window.gtag) return`
   - Fire: `window.gtag('event', 'page_view', { page_path: path, page_title: title })`

3. `interface GA4Item { item_id: string; item_name: string; item_category: string; price: number; quantity: number; }`

4. `trackViewItem(product: { id: string; name: string; price: number; category?: string }): void`
   - Fire `view_item` event with `{ currency: 'USD', value: product.price, items: [one GA4Item] }`

5. `trackAddToCart(item: { id: string; name: string; price: number; category: string; quantity: number }): void`
   - Fire `add_to_cart` event with `{ currency: 'USD', value: item.price * item.quantity, items: [one GA4Item] }`

6. `trackRemoveFromCart(item: { id: string; name: string; price: number; category: string; quantity: number }): void`
   - Fire `remove_from_cart` event with `{ currency: 'USD', value: item.price * item.quantity, items: [one GA4Item] }`

7. `trackBeginCheckout(items: GA4Item[], value: number): void`
   - Fire `begin_checkout` event with `{ currency: 'USD', value, items }`

8. `trackPurchase(orderId: number, value: number, items: GA4Item[], shipping: number, tax: number): void`
   - Fire `purchase` event with `{ transaction_id: String(orderId), value, currency: 'USD', shipping, tax, items }`
   - CRITICAL: Only call this AFTER `paymentIntent.status === 'succeeded'` — never before Stripe confirmation.

Add the global Window type augmentation at the top:
```typescript
declare global {
  interface Window {
    dataLayer: any[];
    gtag: (...args: any[]) => void;
  }
}
```

GA_ID sourced from `import.meta.env.VITE_GA4_MEASUREMENT_ID as string`.

Do NOT install `@types/gtag.js` — the inline `declare global` block is sufficient and avoids a devDependency.

3. Add a third useEffect for Emotive (empty deps `[]` = runs once):
   ```typescript
   // Inject Emotive SMS SDK after 5-second delay
   React.useEffect(() => {
     const emotiveUrl = import.meta.env.VITE_EMOTIVE_SCRIPT_URL as string | undefined;
     const timer = setTimeout(() => {
       // Guard: skip if URL not configured, or if script already injected
       if (!emotiveUrl || document.querySelector(`script[src="${emotiveUrl}"]`)) return;
       const script = document.createElement('script');
       script.src = emotiveUrl;
       script.async = true;
       document.body.appendChild(script);
     }, 5000);
     return () => clearTimeout(timer);
   }, []);
   ```
