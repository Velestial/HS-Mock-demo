**context/CartContext.tsx — add_to_cart:**

1. Add import at top:
   ```typescript
   import { trackAddToCart } from '../utils/analytics';
   ```

2. Inside the `addToCart` function, after the `setItems` call (so we track actual cart changes, not intent), add the tracking call:
   ```typescript
   trackAddToCart({
     id: newItem.id,
     name: newItem.name,
     price: newItem.price,
     category: newItem.category,
     quantity: 1,  // always 1 for a fresh add; CartItem has a category field
   });
   ```
   The `CartItem` interface already has `category: string` — no new fields needed. Place the trackAddToCart call after `setItems(...)` resolves in the function body (outside the setState callback, since the callback may be batched).

   Note: If the item already exists in cart (quantity increment), still fire add_to_cart with quantity 1 — this is the standard ecommerce behavior for "user clicked add again."

**components/widgets/CartSidebar.tsx — remove_from_cart:**

3. Read the existing CartSidebar.tsx to find where `removeFromCart` is called. Add import:

   (Adjust import path depth based on actual file location — CartSidebar is in `components/widgets/`, so analytics is `../../utils/analytics`.)

**components/pages/CheckoutPage.tsx — begin_checkout:**

CheckoutPage is the thin orchestrator that holds `step` state. The `begin_checkout` event should fire when the checkout form is first displayed (i.e., when `step === 'form'` on initial render, not on re-renders from form interactions).

1. Add import:
   ```typescript
   import { trackBeginCheckout } from '../../utils/analytics';
   ```

2. Add a useEffect that fires once when the component mounts:
   ```typescript
   React.useEffect(() => {
     // Map cart items to GA4Item format
     const ga4Items = items.map(item => ({
       item_id: item.id,
       item_name: item.name,
       item_category: item.category,
       price: item.price,
       quantity: item.quantity,
     }));
     trackBeginCheckout(ga4Items, cartTotal);
   }, []); // empty deps — fires once on checkout page mount
   ```
   `items` and `cartTotal` come from `useCart()` which CheckoutPage already calls.

**hooks/useCheckoutSubmit.ts — purchase:**

3. Add import:
   ```typescript
   import { trackPurchase } from '../utils/analytics';
   ```
   (Path: useCheckoutSubmit is in `hooks/`, analytics is `../utils/analytics`.)

4. Inside the `submit` function, find the `if (result.paymentIntent?.status === 'succeeded')` block. After `await updateOrderStatus(order.id, 'processing', result.paymentIntent.id)` succeeds (but before the geocoding try/catch), add:
   ```typescript
   // Fire purchase event ONLY after Stripe confirmed + WC order updated
   const ga4Items = items.map(item => ({
     item_id: item.id,
     item_name: item.name,
     item_category: item.category,
     price: item.price,
     quantity: item.quantity,
   }));
   trackPurchase(order.id, finalTotal, ga4Items, shippingCost, taxAmount);
   ```
   CRITICAL: Do NOT move this call outside the `if (result.paymentIntent?.status === 'succeeded')` block. Firing purchase before payment is confirmed means GA4 over-counts conversions (Pitfall 5 from research).

   The variables `order.id`, `finalTotal`, `shippingCost`, `taxAmount`, and `items` are all already in scope within the `submit` function from the destructured `opts`.
