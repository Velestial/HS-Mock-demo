**server/index.cjs — four changes:**

1. Add requires at top (after existing requires):
```javascript
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');
```

2. Add STRIPE_WEBHOOK_SECRET to REQUIRED_ENV array:
```javascript
const REQUIRED_ENV = ['WC_URL', 'WC_CONSUMER_KEY', 'WC_CONSUMER_SECRET', 'STRIPE_SECRET_KEY', 'STRIPE_WEBHOOK_SECRET', 'FRONTEND_URL', 'NODE_ENV'];
```

3. Mount helmet immediately after `app` is created (before CORS):
```javascript
app.use(helmet());
```

4. Add body size limit to express.json:
```javascript
app.use(express.json({ limit: '10kb' }));
```

5. Create a rate limiter and apply to auth endpoints. Add this block BEFORE the route mounts (after cookieParser):
```javascript
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 10,                   // max 10 attempts per IP per window
  standardHeaders: true,
  legacyHeaders: false,
  message: { error: true, code: 'TOO_MANY_REQUESTS', message: 'Too many attempts. Try again in 15 minutes.' },
});
```

6. Apply the limiter to auth routes specifically. Replace the auth route mount:
```javascript
app.use('/api/auth/login', authLimiter);
app.use('/api/auth/register', authLimiter);
app.use('/api', require('./routes/auth.cjs'));
```

**server/routes/customers.cjs — wrap all async handlers:**

Read the file first. Every `router.get` / `router.put` async handler that does not already have a try-catch needs one added. Pattern for each:

```javascript
router.get('/customer/:id', async (req, res, next) => {
  try {
    // existing body
  } catch (err) {
    next(err);
  }
});
```

Apply to all 4 handlers in customers.cjs.

**server/routes/products.cjs — wrap async handler:**

Same pattern — wrap the single async handler in try-catch with `next(err)`.

**server/routes/orders.cjs — remove PII console.logs:**

Read the file. Find any `console.log(...)` or `console.error(...)` calls that output:
- Full WooCommerce error objects (`wcError`, `wcData`, `response.data`)
- Customer billing/shipping data
- Full `req.body` contents

Replace any such logs with a safe summary that includes only the HTTP status and route name, e.g.:
```javascript
console.error('[create-order] WooCommerce error:', err?.response?.status || 'unknown');
```

Do NOT remove error logging entirely — just strip the payload contents.
