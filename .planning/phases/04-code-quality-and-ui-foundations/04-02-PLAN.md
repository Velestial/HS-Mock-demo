# Plan 04-02: Split CheckoutPage into Step Components

## Context

`components/pages/CheckoutPage.tsx` is ~680 lines handling everything: form state, payment logic, success screen, map rendering. This plan splits it into focused step components.

**Read the current file first** (`components/pages/CheckoutPage.tsx`) before implementing.

## Target Structure

```
components/pages/
  CheckoutPage.tsx          ← Thin orchestrator (~60-80 lines)
  CheckoutFormStep.tsx      ← Form UI: contact + shipping + card + Turnstile (~250 lines)
  CheckoutSuccessStep.tsx   ← Success screen: map + order summary (~180 lines)
```

## Split Design

### CheckoutPage.tsx (orchestrator)
Holds:
- `step` state: `'form' | 'processing' | 'success'`
- `confirmedOrder` state (for passing to success step)
- `mapCoords` state
- `formData` state
- `error` state
- `createdOrderId` ref
- `handleSubmit` logic (createOrder → createPaymentIntent → confirmPayment → updateOrderStatus)
- Renders `<CheckoutFormStep>` or `<CheckoutSuccessStep>` based on step

### CheckoutFormStep.tsx
Receives via props:
- `formData`, `handleInputChange`
- `step` (for button disabled/processing state)
- `error`
- `onSubmit` (the form's handleSubmit)
- `shippingCost`, `taxAmount`, `finalTotal`, `cartTotal`
- `items` (from useCart, for the right-column order summary)
- `turnstileToken`, `setTurnstileToken`

Contains:
- Contact info section
- Shipping details section
- Payment (CardElement + Turnstile)
- Order summary right column
- Submit button

### CheckoutSuccessStep.tsx
Receives via props:
- `confirmedOrder` (items, total, shipping, tax, shippingAddress)
- `mapCoords`
- `formData` (for name/address display)
- `onBack`

Contains:
- Success header with ShieldCheck
- Left column: map + address + shipping schedule disclaimer
- Right column: order summary items + totals
- Return to Base button

## Implementation

1. Read `components/pages/CheckoutPage.tsx` carefully
2. Create `CheckoutFormStep.tsx` — extract the form JSX and its props interface
3. Create `CheckoutSuccessStep.tsx` — extract the success JSX and its props interface
4. Rewrite `CheckoutPage.tsx` as the thin orchestrator that:
   - Holds all state
   - Contains `handleSubmit` logic
   - Renders the appropriate step component
5. Verify `npm run build` passes

## Imports in new files
New files are in `components/pages/` so they use `../../` for external imports:
- `import { useStripe, useElements, CardElement } from '@stripe/react-stripe-js'` (stays as-is, package)
- `import { useCart } from '../../context/CartContext'`
- `import { createOrder, createPaymentIntent, updateOrderStatus } from '../../services/api'`
- `import { Turnstile } from '@marsidev/react-turnstile'` (stays as-is, package)
- `import { MapContainer, TileLayer, Marker, Popup, useMap } from 'react-leaflet'` (stays as-is, package)
