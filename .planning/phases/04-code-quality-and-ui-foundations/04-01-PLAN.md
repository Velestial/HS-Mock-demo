# Plan 04-01: Restructure components/ into Subdirectories

## Context

All 28 components are currently in a flat `components/` directory. This plan moves them into categorized subdirectories and updates all import paths.

## Target Directory Structure

```
components/
  pages/        ← Full-page route destinations (16 files)
  sections/     ← Homepage content sections (5 files)
  widgets/      ← UI overlays, popups, utilities (5 files)
  layout/       ← Navbar, Footer (2 files)
  ui/           ← (empty now, populated by 04-04)
```

## File Assignments

**pages/** (16 files):
AccountPage.tsx, AuthWrapper.tsx, BaitPage.tsx, BundlesPage.tsx, CheckoutPage.tsx, EbooksPage.tsx, FAQPage.tsx, FinalChancePage.tsx, LoginPage.tsx, PrivacyPolicyPage.tsx, ProductPage.tsx, RodPage.tsx, RodWarrantyPage.tsx, ShopPage.tsx, TacklePage.tsx, TermsPage.tsx

**sections/** (5 files):
BentoGrid.tsx, Gallery.tsx, Hero.tsx, ProductDescription.tsx, Reviews.tsx

**widgets/** (5 files):
CartSidebar.tsx, EmotivePopup.tsx, Lightbox.tsx, MobileAddedSuccess.tsx, ScrollToTop.tsx

**layout/** (2 files):
Footer.tsx, Navbar.tsx

## Import Path Changes

After moving, all relative imports in components must update:

### Standard rule (applies to ALL moved files):
- `from '../context/X'` → `from '../../context/X'`
- `from '../types'` → `from '../../types'`
- `from '../services/api'` → `from '../../services/api'`
- `from '../assets/X'` → `from '../../assets/X'`
- `from '../data/products'` → `from '../../data/products'`
- `from '../utils/X'` → `from '../../utils/X'`

### Cross-component imports (special cases):
- `components/pages/AuthWrapper.tsx` imports `./LoginPage` and `./AccountPage` — both are in `pages/`, so these stay as `./LoginPage` and `./AccountPage` ✓ (no change needed)
- `components/pages/ProductPage.tsx` imports `./Lightbox` — Lightbox moves to `widgets/`, so change to `'../widgets/Lightbox'`

## Implementation Steps

1. Create subdirectories: `components/pages/`, `components/sections/`, `components/widgets/`, `components/layout/`, `components/ui/`
2. Use `git mv` to move each file (preserves git history):
   ```bash
   git mv components/BaitPage.tsx components/pages/BaitPage.tsx
   git mv components/Navbar.tsx components/layout/Navbar.tsx
   # etc. for all 28 files
   ```
3. Update imports inside each moved file (the `../` → `../../` rule)
4. Fix the ProductPage.tsx special case (`./Lightbox` → `'../widgets/Lightbox'`)
5. Update App.tsx with all new import paths
6. Run `npm run build` to verify — fix any TypeScript/import errors
