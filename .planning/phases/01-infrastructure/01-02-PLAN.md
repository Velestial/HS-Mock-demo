<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Deploy to TitanHostingHub VPS and verify staging endpoint</name>
  <action>
Human deploys the Express server to TitanHostingHub VPS (or Railway fallback) via SSH. This step requires VPS SSH access that Claude cannot perform.

Before starting: Confirm TitanHostingHub supports persistent Node.js processes. (If denied, use Railway fallback below.)

Steps for TitanHostingHub VPS:

1. SSH into the VPS: `ssh your-user@your-titan-host-ip`

2. Clone or pull the repo on the VPS:
   ```bash
   git clone https://github.com/YOUR_REPO/hey-skipper-mock.git /var/www/hey-skipper
   cd /var/www/hey-skipper
   npm install --production
   ```
   Or if already cloned: `git pull origin main && npm install --production`

3. Edit ecosystem.config.js on the VPS to replace REPLACE_WITH_REAL_* values with real secrets:
   `nano ecosystem.config.js`
   Do NOT push this edited version to git.

4. Install PM2 if not installed: `npm install -g pm2`

5. Start the server in staging mode:
   ```bash
   pm2 start ecosystem.config.js --env staging
   pm2 save
   pm2 startup   # Run the outputted command to enable auto-start on reboot
   ```

6. Verify locally on VPS:
   ```bash
   pm2 list      # Should show hey-skipper-api with status 'online'
   curl http://localhost:3000/api/health
   ```

7. Set up Nginx reverse proxy. Create /etc/nginx/sites-available/api.heyskipper.com:
   ```nginx
   server {
       listen 80;
       server_name api.heyskipper.com;
       return 301 https://$host$request_uri;
   }
   server {
       listen 443 ssl;
       server_name api.heyskipper.com;
       ssl_certificate /etc/letsencrypt/live/api.heyskipper.com/fullchain.pem;
       ssl_certificate_key /etc/letsencrypt/live/api.heyskipper.com/privkey.pem;
       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

8. Enable site and get SSL:
   ```bash
   sudo ln -s /etc/nginx/sites-available/api.heyskipper.com /etc/nginx/sites-enabled/
   sudo certbot --nginx -d api.heyskipper.com
   sudo nginx -t && sudo systemctl reload nginx
   ```

9. Verify from local machine: `curl https://api.heyskipper.com/api/health`
   Expected: `{"status":"ok","timestamp":"..."}`

Railway Fallback (if TitanHostingHub denies Node.js):

3. Set start command: `node server/index.cjs`
4. Verify: `curl https://YOUR-RAILWAY-URL.railway.app/api/health`
